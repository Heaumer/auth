Awesom Auth Server

The AAS is a password-less authentication server. It allows multiple services to share the same user base.
A service can easily authorize its user to use the authentication server of their choice.

Registration

To register, users need

  • A valid email address
  • A nickname
They will then be sent a token which can be used to login to an AAS.

Login

To an AAS

To login, users need to have a token. If they don't, they start by providing either their email or nickname.
An email with a token will be sent to this address.

Eventually, users may use a password. Simply leave the password field blank to let it disabled.

To a service using AAS

Login to services using an AAS should be fairly similar, except:

  • token will be available on the AAS and not via email,
  • users can't login via password
